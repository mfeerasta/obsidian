**Geo mechanism — UPGRADED 2026-06-14 to SERVER-SIDE (authoritative, no leak).** The feerasta-site Worker now reads `request.cf.country` on every HTML request and HTMLRewriter-fills each `.geo-price` element, STRIPPING the other region's data-us/data-ca attr out of the HTML entirely (CA→CAD, else USD). So the other region's price is never in the DOM: no flash, no toggle, nothing to switch to. The old bottom-right US$/CA$ toggle was REMOVED from studio + ascent (it let visitors pick the cheaper region). Requires `run_worker_first = true` in the assets config (else CF serves the static asset before the worker and the transform never runs — this was the bug, fixed). Client `/cdn-cgi/trace` check kept only as fallback if server didn't resolve. Verified both paths live (US shows USD-only, CA shows CAD-only, opposing numbers absent). Applies to pricing.html + studio + ascent + ledger (any page with .geo-price).
